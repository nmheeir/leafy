---
description: Commit Message Generator
---

Based on the staged changes, generate a Conventional Commit message.

Use the last three commit messages as a style reference.

Include:

- A Conventional Commit header

- A commit body written as bullet points (maximum 4 bullet points)

Additional requirements:

- Describe changes at a conceptual or functional level (what changed and why).

- Do not reference file paths, file locations, IDE links, or source code URLs.

- Add one extra bullet point phrased in a way that makes the message easy to copy and reuse.

- Keep the output concise and directly copyable as a commit message.
